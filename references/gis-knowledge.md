## 一、GIS 数据来源

| 数据类型 | 说明 |
|----------|------|
| **地图数据** | 各种普通地图和专题地图，是地理信息的主要载体，内容直观，精度高 |
| **影像数据** | 卫星遥感和航空遥感，能够及时、大范围提供空间信息，对数据库更新极为有利 |
| **数字化测绘数据** | GPS定位、全站仪实测、全数字摄影测量等，可直接获得高精度矢量数据 |
| **实验观测数据** | 直接获取的数字图形和属性数据，需注意数据格式转换和精度可信度 |
| **社会经济数据** | 统计数据、人口数据、经济数据等，可与空间数据关联分析 |
| **文字报告和立法文件** | 城市规划管理等特定类型GIS系统中的重要数据来源 |

## 二、iDesktopX 数据组织结构

### 核心层级

```
工作空间 (Workspace)
├── 数据源 (Datasource)
│   └── 数据集 (Dataset) × N
├── 地图 (Map) × N
├── 场景 (Scene) × N          ← 三维球体环境
├── 布局 (Layout) × N         ← 打印排版
├── 图表 (Chart) × N          ← 属性数据可视化
├── 模型 (Model) × N          ← 自动化处理流程
├── 视频地图 (VideoMap) × N   ← 视频分析
├── 图谱 (KnowledgeGraph) × N ← 知识图谱
├── 大屏 (Dashboard) × N      ← 综合展示
└── 资源 (Resources)          ← 符号库、色系
```

### 工作空间类型

| 类型 | 说明 | 文件扩展名 |
|------|------|-----------|
| 文件型（XML格式）| 轻量，适合个人使用 | `.smwu` |
| 文件型（二进制）| 旧版格式 | `.sxwu`/`.sxw` |
| 数据库型 | 支持多用户协作 | Oracle/PostgreSQL 存储 |

## 三、数据源管理

### 数据源类型

| 类型 | 引擎 | 说明 |
|------|------|------|
| 文件型 | UDB | SuperMap 本地文件型数据源（推荐） |
| 文件型 | UDBX | 支持更大单文件，高性能 |
| 数据库型 | Oracle Spatial | Oracle 空间数据库 |
| 数据库型 | PostgreSQL/PostGIS | 开源空间数据库 |
| 数据库型 | Yukon | 超图自研数据库 |
| Web型 | iServer REST | SuperMap iServer 服务数据源 |
| Web型 | OGC WFS/WMS | 标准 OGC 服务 |

### 核心操作

| 操作 | 说明 |
|------|------|
| 新建数据源 | 创建 UDB/UDBX 文件型数据源或数据库连接 |
| 打开数据源 | 打开已有的本地文件型或远程数据库数据源 |
| 复制数据源 | 将数据源及其所有数据集复制到新位置 |
| 紧缩数据源 | 释放 UDB 文件中的碎片空间（类似磁盘整理） |
| 重新指定 | 修改数据源的存储路径或连接参数 |
| 数据源转换 | UDB ↔ UDBX 互转 |
| 业务表管理 | 将外部数据库表关联到数据源作为属性表 |

### 数据集类型

| 类型 | DatasetType 枚举 | 说明 |

|------|------|

| 重新计算范围 | 重新计算并更新数据集的空间范围 |
| 重新计算面积/周长 | 更新矢量数据集的几何统计字段 |
| 统计栅格值 | 统计栅格数据集的像元值信息 |
| 数据集分组 | 在数据源中对数据集进行分组归类 |
| 属性管理 | 设置字段的值域、子类型、条件值等规则 |
| 编码方式 | 设置数据集压缩编码（如单字节、双字节、LZW等） |

## 六、影像金字塔与镶嵌数据集

### 影像金字塔
- 为栅格/影像数据集创建多分辨率的层级结构
- 目的：加快大影像数据的显示速度
- 操作：创建金字塔 / 删除金字塔 / 查看金字塔信息

### 镶嵌数据集
- 用于管理海量影像数据（如遥感影像图库）
- 支持创建镶嵌数据集，将多张影像组织为统一数据集
- 支持按空间范围、时间等条件查询和显示

---

## 七、流数据管理

- 支持配置实时流数据服务（如 IoT 传感器数据）
- 可发送并接收实时流数据（点、线、面）
- 应用场景：车辆实时轨迹、传感器监测、实时更新地图

## 八、数据版本管理

- 支持数据库（PostGIS、Yukon）中的数据版本控制
- 实现多人协同编辑同一份数据而不冲突
- 功能：注册版本、同步数据、自动存档、查看历史版本

## 十四、空间分析

### 矢量分析

| 分析类型 | 函数 | 说明 |
|----------|------|------|
| 缓冲区分析 | `create_buffer` | 在要素周围创建指定距离的缓冲区 |
| 叠加分析 | `overlay` / `intersect` / `union` / `erase` | 多图层空间关系计算 |
| 裁剪 | `clip_vector` | 用面图层裁剪其他矢量数据 |
| 溶解融合 | `dissolve` | 按属性合并相邻同类面要素 |
| 泰森多边形 | `create_thiessen_polygons` | 基于点集生成 Voronoi 图 |
| 核密度分析 | `kernel_density` | 估算点/线的空间密度分布 |
| 点密度分析 | `point_density` | 统计单位面积内的点数量 |
| 矢量光滑 | `smooth_vector` | 平滑线/面的折角 |
| 矢量重采样 | `resample_vector` | 减少线/面的节点数量 |
| 区域制表 | `tabulate_area` | 统计每个区域内各类别的面积 |
| 属性更新 | `update_attributes` | 按空间关系传递属性值 |

### 栅格分析

| 分析类型 | 函数 | 说明 |
|----------|------|------|
| 核密度分析 | `kernel_density` | 栅格核密度估算 |
| 栅格重采样 | `resample_raster` | 改变栅格分辨率 |
| 栅格重分级 | `reclass_grid` | 按规则重新划分像元值范围 |
| 栅格聚合 | `aggregate_grid` | 降低栅格分辨率并统计像元值 |
| 栅格裁剪 | `clip_raster` | 按范围裁剪栅格数据 |
| 坡度分析 | `calculate_slope` | 计算 DEM 的坡度（0-90°） |
| 坡向分析 | `calculate_aspect` | 计算 DEM 的坡向（0-360°） |
| 插值分析 | `interpolate` | 由离散点插值生成连续栅格面 |

### 插值方法

| 方法 | 适用场景 |
|------|----------|
| IDW（反距离权重）| 通用插值，计算速度快 |
| Kriging | 地统计插值，精度高，考虑空间自相关 |
| 样条函数 | 生成平滑连续曲面 |
| 自然邻域法 | 不规则分布的离散点 |
| 密度分析 | 人口、犯罪等点事件密度估算 |

### 网络分析

| 分析类型 | 函数 | 说明 |
|----------|------|------|
| 最短路径 | `path_line` | 计算两点间最短路径 |
| 服务区分析 | `build_facility_network_directions` | 给定时间/距离内可达区域 |
| 旅行商（TSP）| - | 多站点最优访问顺序 |

## 十六、Python 脚本开发

### Python 窗口使用

1. **打开方式**：开始 → 浏览 → Python
2. **脚本输入窗口**：直接输入代码，按 Enter 执行
3. **执行 Python 文件**：点击"执行Python文件"按钮，选择 `.py` 文件
4. **智能提示**：自动补全方法名和参数
5. **帮助联动**：帮助文档窗口实时显示当前方法文档

### 内置 Python 窗口示例

```python
# 获取当前工作空间（内置环境无需初始化）
from iobjectspy import *

ws = Workspace()
ds = ws.get_datasource(0)
dataset = ds.get_dataset("Points")

# 执行缓冲区分析
result = create_buffer(dataset, 1000, 1000)
print("缓冲区结果:", result)
```
